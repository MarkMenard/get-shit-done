<feature>
  <name>Migration Module: Detection, Migration, Validation</name>
  <files>get-shit-done/bin/lib/migration.cjs, tests/migration.test.cjs</files>
  <behavior>
    detectFlatLayout(cwd):
    - Returns { needs_migration: false } when .planning/.active exists with non-empty slug
    - Returns { needs_migration: false } when .planning/ has none of the known files (PROJECT.md, ROADMAP.md, STATE.md, config.json)
    - Returns { needs_migration: false } when .planning/ does not exist
    - Returns { needs_migration: true, suggested_slug: 'my-project', detected_files: ['PROJECT.md', ...] } when known files exist at .planning/ root and no .active
    - Returns { needs_migration: true, suggested_slug: null, detected_files: ['config.json'] } when PROJECT.md has no title or is missing
    - Any ONE known file is sufficient to trigger detection

    cmdMigrate(cwd, slug, raw):
    - Idempotent: if .active exists with a slug, returns { migrated: false, reason: 'already_namespaced', slug }
    - Rejects reserved slugs that collide with known GSD directories: 'phases', 'milestones', 'codebase', 'config'
    - Creates .planning/<slug>/ directory
    - Moves all entries under .planning/ into .planning/<slug>/ EXCEPT .active, .gitignore, and the slug dir itself
    - Preserves nested structure (phases/01-foo/ stays as <slug>/phases/01-foo/)
    - Updates config.json project_slug at new location
    - Calls writeActiveFile(cwd, slug) to write .active
    - Returns { migrated: true, slug, entries_moved: N, validation: {...} }
    - On any move failure: rolls back all completed moves (renameSync dst->src in reverse), calls error()
    - On missing .planning/ directory: calls error()

    validateMigration(cwd, slug, originalEntries):
    - Checks each original entry exists under .planning/<slug>/
    - Checks planningRoot(cwd) resolves to path ending with slug
    - Returns { valid: true/false, missing: [], planning_root_resolved: true/false }
  </behavior>
  <implementation>
    Create get-shit-done/bin/lib/migration.cjs:
    - require core.cjs for planningRoot, writeActiveFile, safeReadFile, generateSlugInternal, output, error
    - detectFlatLayout(cwd): check .active first (early return false), then scan for known files, derive slug from PROJECT.md title via generateSlugInternal
    - cmdMigrate(cwd, slug, raw): idempotency check, reserved slug check (reject 'phases', 'milestones', 'codebase', 'config', '.gitignore', '.active'), collect entries to move via readdirSync filter, mkdirSync target, move loop with tracking, rollback on catch, update config.json, writeActiveFile, validate, output result
    - validateMigration(cwd, slug, originalEntries): existsSync checks + planningRoot resolution check
    - module.exports = { detectFlatLayout, cmdMigrate }

    Create tests/migration.test.cjs:
    - Use node:test, helpers.cjs patterns (createTempProject, cleanup)
    - Add createTempFlatProject() helper that creates .planning/ with PROJECT.md, ROADMAP.md, STATE.md, config.json, and phases/ subdirectory with a phase dir inside
    - Test detectFlatLayout: 5+ cases (no .planning, empty .planning, .active exists, flat with files, flat without PROJECT.md title)
    - Test cmdMigrate via runGsdTools('migrate <slug>', tmpDir): success case, idempotency, reserved slug rejection, directory structure preservation
    - Test validateMigration indirectly through cmdMigrate output validation field
  </implementation>
</feature>
