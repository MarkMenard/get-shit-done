  <files>get-shit-done/bin/lib/init.cjs, get-shit-done/bin/gsd-tools.cjs</files>
  <action>
    In init.cjs:
    1. Add import at top: `const { detectFlatLayout } = require('./migration.cjs');`
    2. At the START of every `cmdInit*` function (before `planningRoot(cwd)` is called), add the detection check:
       ```
       const migration = detectFlatLayout(cwd);
       if (migration.needs_migration) {
         output(migration, raw);
         return;
       }
       ```
    3. There are ~12 init functions. Each one must get this check. Do NOT use a wrapper/decorator -- add the check explicitly to each function per the established pattern of explicit code in this codebase (no magic). The `raw` parameter is always the last argument in every init function signature.
    4. Special case: `cmdInitNewProject` -- this function creates a NEW project, so detection should NOT block it. Skip adding detection to cmdInitNewProject. A new project on a flat layout should still be able to initialize (migration can happen separately).

    In gsd-tools.cjs:
    1. Add a new case to the command router:
       ```
       case 'migrate': {
         const migration = require('./lib/migration.cjs');
         migration.cmdMigrate(cwd, args[1], raw);
         break;
       }
       ```
    2. Place it alphabetically near the other top-level commands (near 'milestone' or after the existing cases).
  </action>
